# SecAI+ Acronyms (Unofficial / Community-Maintained)

> Unofficial study aid — not affiliated with or endorsed by CompTIA. Align terms with the [exam blueprint](../exam-objectives.md).

---

| Acronym | Expansion — note |
|---|---|
| ABAC | Attribute-Based Access Control — policy by user/resource attributes |
| AI | Artificial Intelligence — machines performing cognitive tasks |
| AIMS | AI Management System — governance system per ISO/IEC 42001 |
| AI RMF | AI Risk Management Framework — NIST: Govern, Map, Measure, Manage |
| AML | Adversarial Machine Learning — attacks/defenses on ML models |
| API | Application Programming Interface — interface to call services/models |
| ATLAS | Adversarial Threat Landscape for AI Systems — MITRE AI attack KB |
| ATT&CK | Adversarial Tactics, Techniques & Common Knowledge — MITRE adversary KB |
| AUP | Acceptable Use Policy — sanctioned-use rules, e.g., for AI tools |
| BCP | Business Continuity Planning — sustaining service during disruption |
| C2 | Command and Control — attacker channel to compromised systems |
| CASP | CompTIA Advanced Security Practitioner — legacy name for SecurityX |
| CIA | Confidentiality, Integrity, Availability — core security triad |
| CSF | Cybersecurity Framework — NIST framework (Identify…Govern) |
| DDoS | Distributed Denial of Service — DoS from many sources |
| DL | Deep Learning — multi-layer neural network ML |
| DLP | Data Loss Prevention — controls to stop sensitive-data exfiltration |
| DoS | Denial of Service — exhausting availability of a system |
| DP | Differential Privacy — noise-based privacy guarantee |
| DPIA | Data Protection Impact Assessment — GDPR privacy risk assessment |
| EU AI Act | European Union AI Act — risk-tiered AI regulation |
| EULA | End User License Agreement — usage rights/terms for software/models |
| GDPR | General Data Protection Regulation — EU data-protection law |
| GenAI | Generative AI — models that create new content |
| GPAI | General-Purpose AI — broad-capability model under EU AI Act |
| GPU | Graphics Processing Unit — parallel hardware for training/inference |
| GRC | Governance, Risk, and Compliance — oversight discipline |
| HITL | Human-in-the-Loop — human approval of AI actions |
| IAM | Identity and Access Management — managing identities/permissions |
| IEC | International Electrotechnical Commission — co-publisher of 42001 |
| IOC | Indicator of Compromise — forensic evidence of an attack |
| IR | Incident Response — detecting and handling security incidents |
| ISO | International Organization for Standardization — standards body |
| JIT | Just-in-Time (access) — temporary, on-demand privileges |
| LLM | Large Language Model — transformer-based text foundation model |
| LLM Top 10 | OWASP Top 10 for LLM Applications — key LLM risks (2025) |
| LLMOps | LLM Operations — deploying/monitoring LLM apps |
| MCP | Model Context Protocol — open standard for tool/data connections |
| MFA | Multi-Factor Authentication — two+ authentication factors |
| ML | Machine Learning — algorithms that learn from data |
| MLBOM | Machine Learning Bill of Materials — inventory of model components |
| ML Top 10 | OWASP Machine Learning Security Top 10 — key ML risks |
| MLOps | Machine Learning Operations — productionizing ML systems |
| NIST | National Institute of Standards and Technology — US standards agency |
| NLP | Natural Language Processing — computational language understanding |
| OBO | On-Behalf-Of — service acts with the user's identity/permissions |
| OECD | Organisation for Economic Co-operation and Development — AI principles |
| OWASP | Open Worldwide Application Security Project — security nonprofit/lists |
| PBQ | Performance-Based Question — hands-on exam task |
| PET | Privacy-Enhancing Technology — e.g., DP, federated learning |
| PII | Personally Identifiable Information — data identifying a person |
| PoLP | Principle of Least Privilege — grant minimum necessary access |
| RAG | Retrieval-Augmented Generation — grounding LLMs with retrieved data |
| RBAC | Role-Based Access Control — permissions assigned by role |
| RCE | Remote Code Execution — running attacker code on a target |
| RL | Reinforcement Learning — learning via reward signals |
| RLHF | Reinforcement Learning from Human Feedback — preference-based alignment |
| RMF | Risk Management Framework — structured risk process (e.g., NIST) |
| SAIF | Secure AI Framework — Google's secure-AI guidance |
| SBOM | Software Bill of Materials — inventory of software components |
| SDLC | Software Development Life Cycle — phases of building software |
| SIEM | Security Information and Event Management — log aggregation/correlation |
| SLM | Small Language Model — compact, efficient language model |
| SOAR | Security Orchestration, Automation, and Response — automated IR |
| SOC | Security Operations Center — team/facility monitoring security |
| SSRF | Server-Side Request Forgery — coercing a server into internal requests |
| TPU | Tensor Processing Unit — Google ML accelerator hardware |
| TTP | Tactics, Techniques, and Procedures — adversary behavior patterns |
| UEBA | User and Entity Behavior Analytics — anomaly detection on behavior |
| XAI | Explainable AI — making model decisions interpretable |
| XSS | Cross-Site Scripting — injecting scripts via untrusted output |
