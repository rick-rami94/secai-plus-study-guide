## Format (mirrors the community blueprint)

- **90 questions**, single best answer (A–D), **165 minutes**.
- Questions are **mixed across domains** (like the real randomized exam) and allocated by weight: **D1 = 14, D2 = 22, D3 = 20, D4 = 16, D5 = 18**.
- Every question is tagged with its objective, e.g., `[2.2]`, so you can map misses back to the [study guide](../study-guide/).

## How to score yourself

1. Take the test **closed-book and timed**. Write down your A/B/C/D for all 90.
2. Open the matching `-answers.md`, use the **answer-key table** at the top to mark right/wrong, then read **every explanation** (including for questions you got right — confirm you got it right for the right reason).
3. Convert: **raw % → community pass guide.** CompTIA scales 100–900 with ~750 to pass; as a rough study target, aim for **≈ 83% correct (about 75/90)** consistently before exam day.
4. Tally your wrong answers **by domain tag** to find your weakest domain, then re-study that chapter.

| Raw correct (of 90) | Approx. % | Read as |
|---|---|---|
| 80+ | 89%+ | Strong — exam-ready in this set |
| 75–79 | 83–88% | Around the community pass line |
| 67–74 | 74–82% | Close; target your weak domains |
| < 67 | < 74% | Keep studying before booking |

## Notes on the answer keys

- Correct answers are **balanced across A/B/C/D** (~25% each) with **no three identical answers in a row** — so you can't game them by pattern, and a run of the same letter is never a hint.
- The same high-yield concepts may appear in more than one variant **at different positions/letters** — that's intentional. **Don't memorize letters; learn the why.**

*These tests are a diagnostic tool. A passing score here is encouraging but not a guarantee — always reconcile with official CompTIA materials.*
