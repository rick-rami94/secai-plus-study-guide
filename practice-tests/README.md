## Format (matches the official exam)

- **60 questions**, single best answer (A–D), **60 minutes**.
- Questions are **mixed across domains** (like the real exam) and allocated by the official weighting: **D1 = 10, D2 = 24, D3 = 14, D4 = 12.**
- Every question is tagged with its objective, e.g., `[2.6]`, so you can map misses back to the [study guide](../study-guide/).

## How to score yourself

1. Take the test **closed-book and timed (60 minutes)**. Record your A/B/C/D for all 60.
2. Open the matching `-answers.md`, use the **answer-key table** to mark right/wrong, then read **every explanation** — including for questions you got right (confirm you got it right for the right reason).
3. The official passing score is **600 on a 100–900 scale** (CompTIA uses scaled, non-linear scoring, so a raw percentage doesn't convert directly). As a **study heuristic**, aim to consistently get **~75%+ correct (about 45/60)** before booking.
4. Tally your wrong answers **by domain tag** to find your weakest domain — then re-study that chapter (most questions come from Domain 2).

| Raw correct (of 60) | Approx. % | Read as (heuristic only) |
|---|---|---|
| 50+ | 83%+ | Strong |
| 45–49 | 75–82% | Likely ready |
| 39–44 | 65–73% | Close; target weak domains |
| < 39 | < 65% | Keep studying before booking |

## Notes on the answer keys

- Correct answers are **balanced across A/B/C/D** (exactly 15 each) with **no three identical answers in a row** — so you can't pattern-game them, and a run of the same letter is never a hint.
- High-yield concepts may appear in more than one variant **at different positions/letters** — intentional. **Don't memorize letters; learn the why.**

*These tests are a diagnostic tool, not a guarantee. Always reconcile with the official CompTIA objectives and materials.*
